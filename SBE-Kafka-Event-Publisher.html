COMPONENTE ARQUITECTURA · EVENT PUBLISHER KAFKA

SBE — Specification by Example

Casos de ejemplo que definen el comportamiento esperado del componente de publicación automática de eventos Kafka.
ID Contexto Ejemplo Resultado Esperado
SBE-EP-001 Método anotado finaliza correctamente Método:
@PublishEvent(topic="alta-solicitada")
Retorno:
new AltaEvent("USR-1") 		El componente serializa el objeto a JSON y lo publica en el topic alta-solicitada de forma asíncrona.
SBE-EP-002 Método anotado devuelve null Método:
@PublishEvent(topic="alta-solicitada")
Retorno:
null 		No se publica ningún evento. No se generan reintentos.
SBE-EP-003 Método anotado lanza excepción Método:
@PublishEvent(topic="alta-solicitada")
Excepción:
RuntimeException 		No se publica ningún evento. No se generan reintentos. El componente registra el error en el logger dedicado.
SBE-EP-004 Error temporal al enviar evento Evento válido, envío falla por timeout.
Reintentos configurados: 3 		El componente reintenta el envío 3 veces de forma asíncrona. Si finalmente se envía, se registra éxito en el logger.
SBE-EP-005 Error permanente al enviar evento Evento válido, Kafka no disponible.
Reintentos configurados: 3 		Tras agotar los reintentos, el evento se envía al dead-letter topic con:
- payload original
- topic original
- número de reintentos
- timestamp
- causa del fallo
SBE-EP-006 Proceso de reintentos desde dead-letter topic Mensaje en dead-letter topic:
{ payload, topicOriginal, causa } 		El componente consume el mensaje y reintenta enviarlo al topic original. Si tiene éxito, registra el resultado en el logger dedicado.
SBE-EP-007 Configuración externa event.publisher.retry.max-attempts=5
event.publisher.dead-letter-topic=eventos-no-enviados 		El componente aplica la configuración sin necesidad de código adicional en el proyecto.
SBE-EP-008 Logging dedicado event.publisher.logger.file=event-publisher.log
event.publisher.logger.level=DEBUG 		El componente escribe logs en consola y en el fichero indicado, en texto plano.
Estos ejemplos sirven como base para derivar ATDD y BDD del componente Event Publisher Kafka.
