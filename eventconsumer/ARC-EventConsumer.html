Arquitectura, Diseño de Componentes y Estructura de Directorios

1. Arquitectura Propuesta (Clean Architecture aplicada a eventos)

La arquitectura se divide en 4 capas bien definidas, desacopladas y orientadas a la extensibilidad:

Objetivo: Crear una infraestructura genérica para publicar y consumir eventos mediante anotaciones, sin acoplarse al broker ni a Spring.

🟦 Capa de Dominio

  • Entidades de dominio
  • Eventos de dominio (POJOs)
  • Interfaces de contratos de publicación y consumo

🟩 Capa de Aplicación

  • Casos de uso
  • Servicios de orquestación
  • Puertos de entrada y salida

🟧 Capa de Infraestructura

  • Anotaciones @PublishEvent y @ConsumeEvent
  • Aspectos AOP
  • Escáner de anotaciones
  • Registro de consumidores
  • Adaptadores del broker (Kafka/Redpanda)
  • Serialización y deserialización

🟫 Capa de Adaptadores

  • Controladores REST
  • Beans que publican eventos
  • Beans que consumen eventos

2. Diseño de Componentes

📌 A. Anotación @PublishEvent

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface PublishEvent {
    String topic();
}

Componente asociado: PublishEventAspect

  • Intercepta métodos anotados
  • Obtiene el retorno
  • Serializa y publica el evento

📌 B. Anotación @ConsumeEvent

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface ConsumeEvent {
    String topic();
    Class<?> eventType();
}

Componentes asociados:

  • ConsumeEventScanner – detecta métodos anotados
  • ConsumerRegistry – registra consumidores por topic
  • EventListenerContainer – gestiona listeners y concurrencia
  • KafkaEventListener – recibe mensajes del broker

📌 C. Adaptadores del Broker

  • KafkaEventPublisher implementa EventPublisher
  • KafkaEventListener recibe mensajes y los delega

📌 D. Serialización

  • EventSerializer
  • EventDeserializer

4. Flujo Completo

📤 Publicación

  • Método anotado con @PublishEvent
  • Aspecto intercepta
  • Serializa retorno
  • Publica en el broker

📥 Consumo

  • Escáner detecta métodos con @ConsumeEvent
  • Registry los registra
  • Listener escucha topic
  • Deserializa mensaje
  • Invoca método consumidor en un hilo del pool
