Feature: US-001 Solicitud de alta de usuario
  Como persona interesada en usar el Simulador de Jubilación
  Quiero enviar una solicitud de alta
  Para poder recibir mis credenciales

  Scenario: Solicitud válida
    Given que el usuario introduce datos personales válidos
    When envía la solicitud de alta
    Then el sistema registra la solicitud
    And devuelve un identificador único
    And se emite el evento "alta-solicitada"

  Scenario: Usuario de acceso duplicado
    Given que existe un usuario con usuarioAcceso "ana80"
    When se envía una solicitud con usuarioAcceso "ana80"
    Then la solicitud es rechazada
    And se envía un email indicando que el usuario de acceso ya existe


Feature: US-002 Activación automática de usuario
  Como sistema de gestión de solicitudes
  Quiero procesar solicitudes recibidas
  Para dar de alta usuarios en estado ACTIVO

  Scenario: Activación correcta
    Given que existe una solicitud pendiente con ID "REQ-123"

  Scenario: Usuario de acceso duplicado

    When el servicio de solicitudes la procesa

    And se envía un email de rechazo

Feature: US-003 Acceso a la aplicación (Login)
  Como usuario ACTIVO
  Quiero acceder con usuario y password
  Para usar la aplicación

  Scenario: Login correcto
    Given que existe un usuario ACTIVO con password correcta
    When introduce credenciales válidas
    Then el sistema devuelve un JWT válido

  Scenario: Usuario inactivo

  Scenario: Usuario bloqueado
    Given que existe un usuario BLOQUEADO

  Scenario: Password incorrecta
    Given que existe un usuario ACTIVO
