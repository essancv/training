ATDD - Simulador de Jubilación

US-001 — Solicitud de alta de usuario

  • CA1: Cuando el usuario envía una solicitud válida, el sistema debe registrar la solicitud y devolver un identificador único.
  • CA2: Si el usuario de acceso ya existe, la solicitud debe ser rechazada y debe enviarse un email informando del rechazo.
  • CA3: El sistema debe emitir el evento alta-solicitada en Kafka con los datos de la solicitud.

US-002 — Activación automática de usuario

  • CA1: Cuando el servicio de solicitudes recibe una solicitud válida, debe crear el usuario en estado ACTIVO.
  • CA2: El sistema debe generar una password aleatoria de 8 caracteres.
  • CA3: Debe enviarse un email al usuario con usuario de acceso, password y URL de acceso.
  • CA4: Si el usuario de acceso ya existe, debe enviarse un email de rechazo.

US-003 — Acceso a la aplicación (Login)

  • CA1: Un usuario ACTIVO con credenciales correctas debe recibir un JWT válido.
  • CA2: Un usuario INACTIVO debe recibir error 403 sin token.
  • CA3: Un usuario BLOQUEADO debe recibir error 403 sin token.
  • CA4: Un usuario con password incorrecta debe recibir error 401.

US-004 — Bloqueo automático por intentos fallidos

  • CA1: Tras un intento fallido, el contador de intentos debe incrementarse.
  • CA2: Tras tres intentos fallidos consecutivos, el usuario debe pasar a estado BLOQUEADO.
  • CA3: Un usuario bloqueado no debe poder iniciar sesión aunque introduzca la password correcta.

US-005 — Solicitud de desbloqueo

  • CA1: Un usuario BLOQUEADO debe poder enviar una solicitud de desbloqueo.
  • CA2: El sistema debe emitir el evento desbloqueo-solicitado en Kafka.
  • CA3: El usuario debe recibir confirmación de que la solicitud ha sido enviada.
  • CA4: Un usuario no bloqueado no debe poder solicitar desbloqueo.

US-006 — Desbloqueo por operaciones

  • CA1: Cuando operaciones recibe un evento desbloqueo-solicitado, debe desbloquear al usuario.
  • CA2: El usuario debe pasar de BLOQUEADO a ACTIVO.
  • CA3: Si el usuario no existe, operaciones debe registrar el error sin modificar estados.

US-007 — Recuperación de password

  • CA1: El usuario debe poder solicitar recuperación de password mediante el endpoint correspondiente.
  • CA2: El sistema debe emitir el evento passrecovery-solicitada
